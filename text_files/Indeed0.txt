OBJECTIFS
Le but du stage est est d'explorer les techniques d'apprentissage automatique (ML) et de traitement du langage naturel (NLP) pour effectuer une analyse statique au niveau du code source, dans le but de détecter les injections de code malveillant.
MISSIONS ET RESPONSABILITÉS
Ce stage se déroulera dans les locaux de SAP Labs France, à Sophia-Antipolis. Les travaux se dérouleront dans le contexte du programme de recherche « Security Research », sur le sujet « Open-Source Security Analysis ». Ce dernier vise à rendre plus sûre la consommation de logiciels open source tout au long du cycle de vie du développement logiciel, tant pour SAP que pour ses clients. Les attaques contre la chaîne d'approvisionnement des logiciels open source ont en effet connu une augmentation en nombre et en sophistication. Dans cette mesure, les attaquants visent à injecter des charges utiles malveillantes dans les composants en aval (c'est-à-dire les paquets) pour infecter les utilisateurs finaux.
Le travail attendu est de rechercher des techniques d'analyse de code statique, avec un accent particulier sur les méthodologies ML/NLP, afin d'analyser les paquets et le code source ouverts pour la détection de logiciels malveillants.
Le stage pourra s’articuler autour des taches suivantes:
Développer des scripts de preuve de concept visant à la détection de code source malveillant.
Appliquer un algorithme NLP pour analyser les caractéristiques du code source.
Entraîner plusieurs systèmes d'apprentissage automatique capables de reconnaître des éléments spécifiques du code source.
Comparer les performances des différents systèmes d'apprentissage automatique développés
Les technologies/techniques impliquées sont : Python, Spark, NLP, et les bibliothèques d'apprentissage automatique.
Nous estimons que 40% du temps sera dédié au développement et 60% aux activités de recherche.
